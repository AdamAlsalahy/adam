# سياسة الأمان - مؤسسة نجمة الإبداع

## مقدمة
هذا المستند يوضح إجراءات الأمان المطبقة على الموقع وأفضل الممارسات لضمان سلامة البيانات والموقع.

## تحسينات الأمان المطبقة

### 1. حماية من هجمات CSRF (Cross-Site Request Forgery)
- ✅ تفعيل CSRF Token على جميع النماذج
- ✅ التحقق من CSRF على جميع طلبات POST
- ✅ حماية الـ Cookies بـ HttpOnly و Secure flags

### 2. حماية من هجمات XSS (Cross-Site Scripting)
- ✅ تفعيل XSS Filter
- ✅ Escaping جميع المدخلات من المستخدم
- ✅ Content Security Policy (CSP)

### 3. حماية من هجمات Clickjacking
- ✅ تفعيل X-Frame-Options: DENY
- ✅ منع عرض الموقع في إطارات خارجية

### 4. حماية من MIME Type Sniffing
- ✅ تفعيل X-Content-Type-Options: nosniff

### 5. التحقق من صحة المدخلات
- ✅ التحقق من جميع المدخلات من المستخدم
- ✅ تحديد الطول الأقصى للمدخلات
- ✅ التحقق من صيغة البيانات (Email, Phone, etc.)

### 6. تشفير البيانات
- ✅ استخدام HTTPS (يجب تفعيله في الإنتاج)
- ✅ تشفير كلمات المرور باستخدام Django's password hashing
- ✅ تشفير الجلسات

### 7. تسجيل الأنشطة (Logging)
- ✅ تسجيل محاولات الوصول غير المصرح
- ✅ تسجيل الأخطاء والاستثناءات
- ✅ تسجيل الأنشطة المريبة

### 8. إدارة الجلسات
- ✅ SESSION_COOKIE_HTTPONLY = True
- ✅ SESSION_COOKIE_SECURE = True (في الإنتاج)
- ✅ CSRF_COOKIE_HTTPONLY = True
- ✅ CSRF_COOKIE_SECURE = True (في الإنتاج)

### 9. حماية من SQL Injection
- ✅ استخدام Django ORM بدلاً من raw SQL
- ✅ استخدام parameterized queries

### 10. حماية من Directory Traversal
- ✅ التحقق من أسماء الملفات
- ✅ تقييد الوصول إلى المجلدات

## إعدادات الأمان في الإنتاج

### يجب تغييرها قبل النشر:

```python
# settings.py

# تعطيل وضع التطوير
DEBUG = False

# تغيير المفتاح السري
SECRET_KEY = 'your-very-long-random-secret-key-here'

# تحديد النطاقات المسموح بها
ALLOWED_HOSTS = ['yourdomain.com', 'www.yourdomain.com']

# تفعيل HTTPS
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True

# تفعيل HSTS
SECURE_HSTS_SECONDS = 31536000  # سنة واحدة
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True

# استخدام قاعدة بيانات قوية
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'database_name',
        'USER': 'database_user',
        'PASSWORD': 'strong_password',
        'HOST': 'localhost',
        'PORT': '5432',
    }
}
```

## أفضل الممارسات الأمنية

### 1. إدارة كلمات المرور
- استخدام كلمات مرور قوية (على الأقل 12 حرف)
- تغيير كلمات المرور بشكل دوري
- عدم مشاركة كلمات المرور
- استخدام مدير كلمات مرور

### 2. تحديث البرامج
- تحديث Django بشكل دوري
- تحديث جميع المكتبات والمتعلقات
- تطبيق patches الأمان فوراً

### 3. النسخ الاحتياطية
- عمل نسخ احتياطية دورية من قاعدة البيانات
- حفظ النسخ الاحتياطية في مكان آمن
- اختبار استعادة النسخ الاحتياطية بشكل دوري

### 4. مراقبة الأمان
- مراقبة ملفات السجلات بحثاً عن أنشطة مريبة
- إعداد تنبيهات للأنشطة غير الطبيعية
- مراجعة السجلات بشكل دوري

### 5. الوصول والصلاحيات
- استخدام مبدأ أقل صلاحية (Principle of Least Privilege)
- إعطاء الصلاحيات الدنيا اللازمة فقط
- مراجعة الصلاحيات بشكل دوري

### 6. حماية البيانات الحساسة
- عدم تخزين بيانات حساسة في الكود
- استخدام متغيرات البيئة
- تشفير البيانات الحساسة

4. **إعطاء الفريق وقتاً** لإصلاح الثغرة قبل الكشف العلني
